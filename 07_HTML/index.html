Separador horizontal

<hr /> genera un separador visible. Este es un ejemplo de tag auto-cerrado que se escribe como <hr> en lugar de <hr></hr>. Docs

Imágenes

<img /> representa una imagen en el documento. Para que se muestre, tenemos que indicar donde se encuentra almacenada a través del atributo "src" y la dirección de la imagen.

Por ejemplo, este tag:
<img src="https://upload.wikimedia.org/wikipedia/commons/6/61/HTML5_logo_and_wordmark.svg" />

Enlaces

<a> crean hipervínculos en una página. Requieren al menos el atributo "href" con el que pasamos la dirección a la que este enlace debe dirigir, similar a cómo lo hacemos con el "src" de las imágenes.

Por ejemplo, este código: <a href="https://github.com/">Ir a Github</a>

Listas

<ul> and <li> se usan para generar listas no enumeradas en la página.

Por ejemplo, este código:

<ul>
  <li>Lunes</li>
  <li>Martes</li>
  <li>Sábado</li>
</ul>

Genera esta lista:

  • Lunes
  • Martes
  • Sábado

También podemos hacer listas numeradas cambiando ul por ol al momento de generarla:

<ol>
  <li>Lunes</li>
  <li>Martes</li>
  <li>Miércoles</li>
</ol>
  1. Lunes
  2. Martes
  3. Miércoles

Tablas

<table> se usa para indicar que la información se va a mostrar de forma tabular. Este tag requiere de otros como <tr>, <td> <th> para estructurar nuestra tabla.

Por ejemplo, con este código:

<table border="1">
  <tr>
    <th>Marca</th>
    <th>Modelo</th>
    <th>Potencia</th>
  </tr>
  <tr>
    <td>Tesla</td>
    <td>S</td>
    <td>1,052 hp</td>
  </tr>
  <tr>
    <td>Bugatti</td>
    <td>Divo</td>
    <td>1,500 hp</td>
  </tr>
   <tr>
    <td>Pagani</td>
    <td>Huayra</td>
    <td>850 hp</td>
  </tr>
  <tr>
    <td>Toyota</td>
    <td>GR010 Hybrid</td>
    <td>939 hp</td>
  </tr>
</table>

Produces this table:

Marca Modelo Potencia
Tesla S 1,052 hp
Bugatti Divo 1,500 hp
Pagani Huayra 850 hp
Toyota GR010 Hybrid 939 hp

Divisiones

<div> son contenedores genéricos de elementos que sirven para agrupar estilos, acciones y dar estructura.

Hay otros contenedores con significado semántico como artículos or navs o footers que se comportan exactamente como div, pero con un nombre distinto que nos ayuda a entender qué es lo que está contenido cuando estamos leyendo o limpiando el código, entre otras razones. Docs

Strong

<strong> le dan un aspecto de negritas al texto que contienen, para hacerlo simplemente encapsulamos el texto así y <strong>este texto estaría en negritas</strong>. El uso no es solo para el estilo, tiene un significado semántico de urgencia o importancia que el browser y otros programas que pueden leer HTML puede utilizar. Docs
